# 智能抽油烟机系统 - 需求文档

## 1. 项目概述

### 1.1 项目名称
基于STM32H562VIT6 + OpenMV的端侧AI智能抽油烟机系统

### 1.2 项目目标
设计一款具备环境感知、智能决策、视觉辅助的抽油烟机控制系统，利用端侧AI实现油烟/颗粒物自动检测与风机调速、人员检测与自动开关、滤网寿命预测等功能。

### 1.3 系统组成
- **主控**: STM32H562VIT6 (Cortex-M33, 250MHz, 内置DSP/FPU)
- **视觉协处理器**: OpenMV (STM32H743VI, Cortex-M7, 480MHz)
- **传感器组**: SHT40, SGP40, PMS7003, 霍尔电流传感器

## 2. 功能需求

### FR-01: 多传感器环境感知
| ID | 传感器 | 测量参数 | 接口 | 采样率 |
|----|--------|---------|------|--------|
| FR-01.1 | SHT40 | 温度(-40~125°C), 湿度(0~100%RH) | I2C | 1 Hz |
| FR-01.2 | SGP40 | VOC指数(0~500) | I2C | 1 Hz |
| FR-01.3 | PMS7003 | PM1.0, PM2.5, PM10 (μg/m³) | UART | 1 Hz |
| FR-01.4 | 霍尔电流 | 电机电流(0~10A) | ADC | 100 Hz |

### FR-02: 智能风机控制
- FR-02.1: 根据传感器融合结果自动调节风机转速 (0-5档)
- FR-02.2: 支持手动/自动模式切换
- FR-02.3: PWM驱动，频率20kHz，占空比0-100%
- FR-02.4: 电机过流保护 (霍尔电流 > 8A 触发)

### FR-03: 端侧AI推理
- FR-03.1: **传感器融合模型** (运行于STM32H562): 多传感器输入 → 空气质量等级/风机档位预测
  - 输入: [温度, 湿度, VOC指数, PM1.0, PM2.5, PM10, 电流, 电流变化率] × 时间窗口
  - 输出: 风机档位 (0-5) + 空气质量类别 (优/良/中/差)
- FR-03.2: **烟雾视觉检测** (运行于OpenMV): 实时检测烹饪烟雾
- FR-03.3: **人员检测** (运行于OpenMV): 检测灶台前是否有人
  - 无人超过3分钟 → 自动关机
- FR-03.4: **手势识别** (运行于OpenMV): 隔空手势控制风机档位

### FR-04: 健康与安全
- FR-04.1: 空气质量分级显示 (LED指示灯/OLED屏)
- FR-04.2: 滤网使用寿命预测 (基于累计颗粒物与运行时间)
- FR-04.3: 异常检测与报警 (传感器失效、电机堵转、过温)

### FR-05: 通信与扩展
- FR-05.1: 主控 ↔ OpenMV: UART 921600bps, 自定义协议
- FR-05.2: 预留WiFi/BLE模块接口 (UART2)
- FR-05.3: 预留OLED显示接口 (I2C2)

## 3. 非功能需求

### NFR-01: 实时性
- 传感器采集周期: 1s
- AI推理周期 (STM32H5): ≤ 100ms
- OpenMV推理周期: ≤ 200ms
- 电机响应延迟: ≤ 500ms

### NFR-02: 可靠性
- 传感器通信故障自动重试3次，失败则报警
- 看门狗定时器 (IWDG) 防止系统死锁
- 电机过流硬件保护 + 软件保护双重机制

### NFR-03: 功耗
- 待机模式: < 2W (仅保留人员检测)
- 全速运行: < 15W (含电机)

### NFR-04: 端侧AI约束
- 模型大小: Flash占用 < 256KB (STM32H562有1MB Flash)
- RAM占用: < 128KB (STM32H562有384KB SRAM)
- 推理延迟: < 100ms (Cortex-M33 DSP加速)

## 4. 开发阶段

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| Phase 1 | 硬件接口验证 | 传感器驱动、PWM控制、ADC采样 |
| Phase 2 | 基础控制逻辑 | 传感器数据采集、手动风机控制 |
| Phase 3 | AI模型训练 | 数据集构建、模型训练与量化 |
| Phase 4 | 端侧部署 | STM32Cube.AI部署、OpenMV模型部署 |
| Phase 5 | 系统集成 | 主控-OpenMV联调、全功能测试 |
| Phase 6 | 优化迭代 | 推理延迟优化、功耗优化 |
